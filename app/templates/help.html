What does "LIN" stands for ?

LIN is the acronym of the application's name and stands for "Lost In Network".

What is the main goal of LIN ?

The main goal of LIN is to provide an easy and user-friendly way of managing your networked devices and check for security flaws on them. LIN can also act as a backup service for your configuration files, although it is not the main goal of the application.

What can LIN actually do ?

With LIN you can save your networked devices to a database. This allows an easier management of what equipment is present on the network. All passwords are stored in the database and encrypted with an AES cipher, and are only accessible to a logged-in user. You can use LIN to store and remember the passwords for your devices. Each device can be linked to a lan and can have several other information.

One of the main features of LIN is that it can automatically connect to your devices and fetch your configuration files for backup and analysis purpose. It also checks for CVE flaws on your devices kernel version and warns you when a security issue is detected.

Here is a non-exhaustive list of LIN's capabilities :
  • Store a list of devices
  • Store the passwords in a secure way for each device
  • Perform CVE detection on devices
  • Update the CVE database from the user interface
  • Backup your configuration files in a secure way
  • Provide a way to download your configuration files (encrypted and not encrypted)
  • Allow to check for system information online

Why are there multiple steps to add a device ?

The separated steps, such as creating the device type or the lan, allows us to provide a reusability of the information stored in the database. This way the database doesn't contain any redundancy and thus optimizes the database usage.

Are the data stored in the database safe ?

All sensible information stored in the database such as passwords is encrypted in a way that no external user can read them. They are ecrypted using an AES cipher using the main user's password SHA512. Once the user log in, this hash is stored in the session which is itself encrypted by the Flask micro-framework. This way only a logged-in user can access sensible data and LIN doesn't store the password itself, not even the hash. Session is the main key in the security process on LIN.

The configuration files stored on the server are also encrypted this way. This can explain why changing your password can take some time when the database contains a lot of devices, the key to the AES cipher changes so LIN has to decrypt all the devices and files using the old password before encrypting them again with the new one.
